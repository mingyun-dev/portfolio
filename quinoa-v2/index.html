Multi-underlying asset 기반의 Vault 구현

Vault는 유저들이 예치한 토큰들을 실질적으로 관리하는 주체입니다. Vault 컨트랙트 중 Rebalancing 로직 외에 모든 부분을 담당하여 개발하였습니다.

  1. Chainlink의 Price Feed 컨트랙트 연동

    여러 종류의 Underlying-asset 토큰들을 달러 가격 기준으로 통합하여 관리

    Vault의 수익률과 Share 토큰의 가격 역시 달러 가격 기준으로 책정

  1. Quickswap 컨트랙트를 이용한 토큰 스왑 기능

    유저는 Underlying-asset 중 자신이 원하는 토큰으로 인출 가능

    유저가 자신의 비중만큼 인출을 진행할 때, 여러 종류의 토큰들 사이에서 스왑 기능 이용

다른 DeFi 프로토콜과 연동된 Strategy 구현

Vault에서 관리하는 토큰마다 해당 토큰을 기반으로 하는 Strategy가 존재하며, Strategy로 이동한 토큰들은 다른 DeFi 프로토콜에 예치되어 추가 Yield 수익을 얻습니다.

Stargate, Beefy, Balance 등 DeFi 서비스와 연동하여 Strategy 컨트랙트 상에서 외부 DeFi Vault에 자금을 예치하고 인출할 수 있도록 하였습니다.

Ex. USDC 토큰 기반의 Strategy Logic

function deposit() external { 
        uint256 underlyingAmount = _availableUnderlyings();
        if(underlyingAmount > 0) {
            uint256 poolId = IStargatePool(yieldPool).poolId();

            IERC20(underlyingAsset).approve(yield, underlyingAmount);
            IStargateRouter(yield).addLiquidity(poolId, underlyingAmount, address(this));
        }

        uint256 starAmount = IStargatePool(yieldPool).balanceOf(address(this));
        if(starAmount > 0) {
            IERC20(yieldPool).approve(delegate, starAmount);
            IBeefyVault(delegate).depositAll();
        } 
    }
  • 먼저 Stargate의 USDC Pool에 토큰을 예치
  • 그 결과로 받은 Star USDC 토큰을 Beefy에 재예치
  • 이를 통해 Rebalancing을 이용한 분산 투자와 더불어 각 토큰마다 부가적인 Yield 수익 획득

Transaction의 event log를 활용하여 원하는 값 파싱

Vault에 토큰을 예치한 유저들에게 예치 시점의 share 토큰의 평단가와 현재 수익률을 Quinoa 프론트 화면에서 보여줍니다.

이를 위해 스마트 컨트랙트에서 Share 토큰의 가격과 발행된 개수 등의 정보를 담아 Deposit 이벤트를 발생시키고, 프론트에서는 유저의 Address와 Block Number 등을 활용하여 Polygon 메인넷의 트랜잭션을 필터링하도록 하였습니다.

이후 유저의 Deposit 이력과 Withdraw 이력을 통해 평단가를 계산하였습니다.

성장 경험

PR Template을 통한 코드 리뷰 문화 도입

팀원들마다 어떤 테스크를 맡고 있는지 알고 있어도 실제 코드 리뷰를 진행할 때에는 타인이 작성한 코드의 의미를 빠르게 이해하기에는 무리가 있다고 판단했습니다.

따라서 PR 용 Template을 통해 자신이 맡은 테스크에 대한 간단한 설명과 코드 리뷰 때 중점적으로 봐주었으면 하는 부분이 있는지 설명하는 것을 제안하였습니다. 코드 리뷰를 진행하기 전 설명을 읽으니 이전보다 더 효율적으로 코드 리뷰를 진행할 수 있었고, 추가적인 자료를 첨부해 다른 팀원들의 개발 진행 상황을 더 잘 파악할 수 있었습니다.

비개발자 팀원과의 협업

Quinoa 팀은 개발자 3명과 디자인 1명으로 구성되어 있으며, 4명 모두 기획과 서비스 디자인에 참여하였습니다. 프로젝트를 만드는 빌더가 아니라 “유저”의 입장에서 DeFi 투자에 있어서 필요한 정보들이 무엇인지 고민하면서 서비스를 발전시켰습니다.

또한 디자이너 팀원과 프론트 퍼블리싱 코드 공유를 위해 Git을 활용하였습니다. 해당 팀원은 Git을 활용하는 것에 익숙하지 않아 팀원의 Git에 대한 이해를 돕기 위해 전체적인 개념을 설명하기 위해 내부 문서를 작성하고 공유하였습니다.

Code size를 고려한 Smart contract 개선

스마트 컨트랙트의 복잡도가 높아지면서 Code size 제약에 빈번하게 제약을 받기 시작했습니다.

처음에는 Solidity 컴파일러의 optimizer 설정을 바꾸어 가면서 조절했으나, 이것이 막연히 좋은 해결책이 아님을 깨닫고 다른 방법을 모색하였습니다. 그 결과 스마트 컨트랙트의 로직 개선, library 활용, Custom Error 등을 활용하여 Code size 개선에 성공하였습니다.
